# Провайдеры, модели и API-ключи

Этот файл объясняет, как выбрать сервис перевода, чем отличаются основные настройки модели и как не запутаться в ключах.

## Какие провайдеры поддерживаются

По текущей конфигурации проекта в программе видны, например:

- `Google Gemini Free`;
- `Deepseek API`;
- `Perplexity API MOD`;
- `OpenRouter`;
- `Hugging Face`;
- локальные модели `Ollama/LM Studio`;
- `DeepSeek Web (FreeDeepseekAPI)`;
- `NeuroAPI`;
- `ChatGPT Web (work_ascii)`.

Состав может меняться через конфигурацию программы.

## Как выбрать провайдера

Ориентируйтесь не на “самый модный”, а на задачу:

- если нужен быстрый старт, берите тот сервис, где у вас уже есть рабочий ключ;
- если важна стабильность длинных прогонов, избегайте слишком жёстких бесплатных лимитов;
- если хотите максимум локального контроля, пробуйте `Ollama/LM Studio`;
- если нужен браузерный сценарий, смотрите на `ChatGPT Web`.

## Работа с API-ключами

В интерфейсе управления ключами обычно можно:

- выбрать сервис;
- добавить ключи вручную;
- загрузить ключи из `.txt`;
- удалить отдельные ключи;
- удалить все ключи провайдера;
- удалить вообще все ключи.

Практический смысл:

- несколько ключей помогают переживать лимиты и распределять нагрузку;
- ключи лучше хранить по сервисам, а не смешивать в одном списке без структуры.

## Как хранить ключи удобно

Хорошая практика:

- один ключ на строку;
- отдельный файл под каждого провайдера;
- не мешать тестовые и рабочие ключи;
- не удалять удачные ключи только потому, что один раз был сетевой сбой.

## Главные настройки модели

Пользователь чаще всего сталкивается с такими параметрами:

- модель;
- RPM;
- параллельные запросы;
- RPD или суточный лимит;
- температура;
- `Thinking`;
- системная инструкция.

## Что означают настройки простыми словами

- `RPM` — сколько запросов в минуту вы пытаетесь делать.
- `Параллельные запросы` — сколько задач программа отправляет одновременно.
- `RPD` — дневной лимит по проекту или ключу.
- `Температура` — насколько ответ свободный и вариативный.
- `Thinking` — дополнительный режим размышления, если провайдер поддерживает его.
- `Системная инструкция` — базовое правило поведения модели.

## Как выставлять настройки без боли

Если вы не знаете, с чего начать:

- ставьте умеренный RPM;
- начинайте с небольшой параллельности;
- не поднимайте температуру без причины;
- не включайте сложный режим мышления, пока не убедились, что он реально помогает именно вашей задаче;
- меняйте только один важный параметр за раз.

## Когда поднимать параллельность

Увеличение параллельности полезно, если:

- сервис стабилен;
- лимиты позволяют;
- качество уже устраивает;
- вам действительно нужно ускорение.

Не стоит поднимать её, если:

- идут сетевые ошибки;
- ключи часто упираются в лимиты;
- провайдер нестабилен;
- вы ещё не проверили качество на малом объёме.

## Системный промпт

Редактор системных промптов нужен, когда вы хотите:

- усилить буквальность;
- зафиксировать стиль;
- явно запретить сокращения и пропуски;
- потребовать единый формат ответа.

Но стоит помнить:

- агрессивный промпт может ухудшить естественность текста;
- избыточный контроль иногда ломает ответ модели сильнее, чем помогает.

## Динамический глоссарий

В некоторых настройках есть параметры динамического глоссария и fuzzy-поиска. Они нужны для того, чтобы модель лучше цеплялась за термины из проекта.

Для пользователя это означает:

- функция полезна на книгах с большим количеством имён и понятий;
- первый прогон может быть медленнее из-за калибровки;
- потом работа становится комфортнее.

## Локальные модели

Если вы используете `Ollama` или `LM Studio`:

- программа может подтягивать список моделей с локального сервера;
- скорость и качество зависят уже не только от программы, но и от вашей машины;
- не ждите, что тяжёлая модель на слабом железе даст комфортную скорость.

## DeepSeek Web через FreeDeepseekAPI

Провайдер `DeepSeek Web (FreeDeepseekAPI)` подключается к локальному proxy [ForgetMeAI/FreeDeepseekAPI](https://github.com/ForgetMeAI/FreeDeepseekAPI). Он не требует API-ключа в переводчике: proxy использует ваш сохранённый login DeepSeek Web.

Минимальный запуск proxy:

```powershell
git clone https://github.com/ForgetMeAI/FreeDeepseekAPI.git
cd FreeDeepseekAPI
npm run auth
npm start
```

То же самое можно сделать из GUI без отдельной консоли: выберите провайдер `DeepSeek Web (FreeDeepseekAPI)`, нажмите кнопку `DeepSeek` рядом со списком моделей, укажите папку репозитория FreeDeepseekAPI, затем используйте `Войти в DeepSeek` и `Запустить proxy`. Путь к папке сохраняется в настройках сессии.

По умолчанию переводчик ждёт proxy на `http://127.0.0.1:9655/v1/chat/completions`. Если proxy запущен, список моделей подтянется из `/v1/models`; если нет, останутся встроенные aliases: `deepseek-chat`, `deepseek-reasoner`, search-варианты и expert mode.

Для первого теста лучше выбрать `DeepSeek Chat (FreeDeepseekAPI)`, поставить параллельность `1` и прогнать одну небольшую главу. Это web-proxy, поэтому стабильность зависит от текущей сессии DeepSeek Web; при `401/403/429` обновите login через кнопку `DeepSeek` или команду `npm run auth`.

## ChatGPT Web

Для режима `ChatGPT Web (work_ascii)` есть отдельные настройки:

- workspace name;
- workspace index;
- таймауты;
- шаблон свежего профиля;
- частота обновления профиля;
- вход и регистрация в ChatGPT.

Этот режим требует более аккуратной настройки окружения, чем обычные API.

## Debug-логи

Если что-то работает странно, полезны debug-логи:

- можно открыть папку логов;
- можно фильтровать операции;
- можно ограничить объём логов.

Обычному пользователю это нужно не постоянно, а только при разборе проблем.

## Рекомендуемый минимальный старт

Для первого проекта используйте такую стратегию:

1. Один стабильный провайдер.
2. Одна основная модель.
3. Небольшая параллельность.
4. Простой промпт без перегруза.
5. Тест на 1–2 главах.

Только после этого имеет смысл ускорять пайплайн.
